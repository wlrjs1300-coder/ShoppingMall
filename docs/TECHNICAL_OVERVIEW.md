# 기술 구조 설명

## 프로젝트 형태

이 프로젝트는 서버 없이 동작하는 정적 웹 프로젝트입니다.

- 화면: HTML
- 디자인: CSS
- 기능: JavaScript
- 데이터 저장: 브라우저 `localStorage`
- 관리자 임시 접근 확인: 브라우저 `sessionStorage`

## 주요 파일 역할

| 파일 | 역할 |
| --- | --- |
| `index.html` | 고객용 메인 화면 |
| `menu.html` | 메뉴 목록, 검색, 필터, 주문 상담 요청 |
| `faq.html` | 자주 묻는 질문 |
| `admin.html` | 관리자 ERP 화면 |
| `styles.css` | 전체 레이아웃, 반응형, 컴포넌트 스타일 |
| `script.js` | 메뉴 기능, 주문 저장, 관리자 ERP 로직 |

## 데이터 저장 구조

현재 데이터는 브라우저에 저장됩니다. 같은 브라우저에서 새로고침해도 유지되지만, 다른 PC나 다른 브라우저와 공유되지는 않습니다.

| 저장 키 | 용도 |
| --- | --- |
| `tteokOrders` | 주문 상담 요청과 관리자 직접 등록 주문 |
| `tteokCustomerNotes` | 주문 기반 고객 메모 |
| `tteokCustomers` | 관리자가 직접 등록한 고객 |
| `tteokInventory` | 원재료 재고와 안전 재고 |
| `tteokRecipes` | 상품별 원재료 배합 기준 |
| `tteokInventoryLogs` | 생산 완료 시 원재료 사용 이력 |
| `tteokPurchaseOrders` | 원재료 발주 요청과 진행 상태 |
| `tteokSuppliers` | 공급처 연락처, 취급 품목, 거래 메모 |
| `tteokActivityLogs` | 주문, 생산, 재고, 발주 주요 활동 기록 |
| `tteokAdminAccess` | 관리자 확인 코드 통과 여부 |

## 고객 주문 흐름

1. 고객이 `menu.html`에서 상품을 선택합니다.
2. 주문 상담 요청 패널에 수량, 날짜, 시간, 연락처, 이름을 입력합니다.
3. 수령 방식으로 매장 픽업 또는 배송 상담을 선택합니다.
4. 배송 상담 선택 시 배송 주소 입력이 필요합니다.
5. 제출된 주문은 `tteokOrders`에 저장됩니다.
6. 관리자는 `admin.html` 주문관리에서 해당 주문을 확인합니다.

## 관리자 ERP 흐름

관리자 첫 화면의 운영 알림 센터는 주문, 재고, 발주, 물류 데이터를 다시 읽어 처리 대기 수와 대표 항목을 표시하고 각 관리 탭으로 연결합니다.

최근 활동 로그는 주요 변경 시점에 카테고리, 내용, 관련 탭, 처리 시간을 저장하며 최근 6건을 관리자 첫 화면에 표시합니다.

### 주문관리

- 주문 검색
- 상태 변경
- 판매가와 원가 입력
- 주문 직접 등록, 수정, 삭제
- CSV 저장
- JSON 백업/복원

### 고객관리

- 주문 데이터 기반 고객 자동 집계
- 고객 직접 등록, 수정, 삭제
- 고객 메모 관리

### 생산관리

- 픽업일과 상품 기준으로 생산 항목 집계
- 생산 수량, 주문건수, 픽업시간 확인
- 상품명과 수량 기준 예상 원재료 표시
- 준비완료 일괄 처리
- 준비완료 처리 시 `tteokInventory` 재고 자동 차감

### 재고관리

- 원재료 품목 등록, 수정, 삭제
- 현재 재고와 안전 재고 기준으로 부족/주의/정상 상태 표시
- 생산관리에서 사용하는 대표 원재료 기준표와 연동
- 재고관리 화면에서 상품별 원재료 기준표 표시, 추가, 수정, 삭제
- 생산 완료 시 원재료 사용 이력 자동 기록
- 안전 재고 기준 부족·주의 품목의 권장 발주량 계산
- 권장 발주량으로 발주 요청 생성
- 발주요청, 발주중, 입고완료 상태 관리
- 발주별 공급처와 단가 저장
- 발주 수량과 단가 기준 품목별 발주금액 및 진행 발주액 계산
- 공급처 등록, 수정, 삭제와 발주 입력 자동완성 연동
- 입고완료 처리 시 현재 재고 즉시 갱신
- 기본 재고 샘플 생성

### 픽업/배송 관리

- 매장 픽업과 배송 상담 주문 구분
- 물류 상태 변경
- 완료 처리 시 주문 상태를 픽업완료로 연동

### 매출관리

- 상품별 매출, 원가, 이윤, 이윤율 계산
- 일자별 매출 요약
- 매출 1위, 이윤 1위, 원가 미입력 건수 확인
- 기간 내 입고완료 발주비와 공급처별 비용 집계
- 순현금흐름은 총매출에서 입고 발주비를 차감해 표시
- 예상이윤은 주문 원가 기준이며 순현금흐름과 별도 지표로 유지
- 주문 생성월과 발주 입고완료월을 기준으로 최근 12개월 회계 추이 집계
- 현재 회계 기간의 월별 집계 결과를 UTF-8 BOM CSV로 저장

## 백업 구조

관리자 화면의 백업 저장 기능은 JSON 파일을 생성합니다.

백업에 포함되는 데이터:

- 주문 데이터
- 고객 메모
- 직접 등록 고객
- 원재료 재고
- 원재료 배합 기준
- 원재료 사용 이력
- 원재료 발주 기록
- 공급처 정보

CSV 저장 기능은 주문 목록을 엑셀에서 열 수 있는 형태로 저장합니다.

## 현재 보안 수준

관리자 확인 코드는 임시 접근 확인입니다.

- 현재 코드: `4525`
- 저장 방식: `sessionStorage`
- 목적: 실수로 관리자 화면이 바로 노출되는 것 방지
- 잠금 방식: `관리 잠금` 버튼을 누르면 `tteokAdminAccess` 값을 제거하고 코드 입력 화면으로 복귀

실제 운영에는 서버 로그인, 비밀번호 암호화, 권한 관리가 필요합니다.

## 실제 운영 전 필요한 개발

- 서버와 데이터베이스 연결
- 관리자 로그인 구현
- 네이버 스마트스토어 또는 결제 시스템 연동
- 주문 알림 문자/카카오 알림 연동
- 실제 배송/송장 관리
- 주문·고객·재고·발주·공급처 관리자 통합 검색
