# 인증 운영 정책

## 확정된 방식

- 휴대폰 인증: 운영 환경에서는 Solapi SMS를 사용한다. `SOLAPI_API_KEY`, `SOLAPI_API_SECRET`, `SOLAPI_SENDER_PHONE` 중 하나라도 없으면 인증 요청을 503으로 차단한다.
- 인증번호 저장: 원문 대신 `AUTH_CODE_PEPPER`를 이용한 HMAC-SHA256 해시만 저장한다. 24시간이 지난 소비 기록과 만료 기록은 다음 발송 요청 때 정리한다.
- 로그인 잠금: 고객 계정은 5회 연속 실패하면 15분간 잠근다. 존재하지 않는 계정, 비밀번호 오류, 잠긴 계정은 동일한 메시지를 반환한다.
- 비밀번호 찾기: 공개 기능으로 유지한다. 운영 환경에서는 Resend로 15분짜리 일회성 링크를 발송한다. 계정 존재 여부와 발송 성공 여부는 공개 응답에서 구분하지 않는다.
- 개발·테스트 환경도 인증번호와 재설정 토큰 원문을 API 응답이나 로그에 표시하지 않는다. 자동 테스트는 테스트 전용 고정값을 프로세스 환경에 주입한다.
- 소셜 로그인: 소셜 계정으로 신규 회원을 자동 생성하지 않는다. 인증된 이메일과 같은 이메일로 일반가입한 활성 계정이 있을 때만 소셜 연결을 생성한다.

## 배포 환경변수

필수 보안값과 서비스 키는 `server/.env.example`을 기준으로 배포 플랫폼의 비밀 환경변수에 등록한다. `.env` 파일을 저장소에 커밋하지 않는다.

소셜 콜백 주소:

- Google: `{PUBLIC_BASE_URL}/api/auth/social/google/callback`
- Kakao: `{PUBLIC_BASE_URL}/api/auth/social/kakao/callback`
- Naver: `{PUBLIC_BASE_URL}/api/auth/social/naver/callback`

실제 도메인이 확정되면 각 제공자 개발자 콘솔에 위 주소를 등록하고 운영 앱 키를 배포 환경변수에 입력한다.
