# 따뜻한 떡집 발표 가이드

## 발표 핵심 문장

따뜻한 떡집 프로젝트는 떡 판매 홈페이지와 매장 운영용 ERP 화면을 함께 구성한 쇼핑몰 초기 모델입니다. 고객은 메뉴를 확인하고 주문 상담 요청을 남기며, 관리자는 주문, 고객, 생산, 픽업/배송, 매출 흐름을 한 화면에서 관리할 수 있습니다.

## 발표 순서

### 1. 메인 화면

- 브랜드명과 로고 확인
- 따뜻한 떡집의 첫인상과 주요 메뉴 이동 확인
- 과하지 않은 떡집 분위기의 디자인 방향 설명

추천 멘트:
메인 화면은 고객이 처음 접하는 화면이기 때문에 상품보다 브랜드 분위기와 메뉴 이동을 우선했습니다.

### 2. 메뉴 화면

- BEST 상품 확인
- 전체 메뉴 검색, 카테고리 필터, 페이징 확인
- 상품 클릭 후 상세 안내 확인
- 주문 상담 요청 작성

추천 멘트:
메뉴 페이지는 단순 메뉴판이 아니라 고객이 바로 상담 요청을 남길 수 있도록 구성했습니다.

### 3. 주문 문의 흐름

- 메뉴 선택
- 수량, 날짜, 시간, 연락처 입력
- 매장 픽업 또는 배송 상담 선택
- 배송 상담 선택 시 배송 주소 입력 필요 확인

추천 멘트:
고객이 입력한 문의 정보는 브라우저 저장소에 저장되고 관리자 페이지에서 바로 확인할 수 있습니다.

### 4. 관리자 접속

- `admin.html` 접속
- 확인 코드 `4525` 입력
- 관리자 대시보드와 운영 흐름 카드 확인
- 회원, 생산, 물류, 회계 ERP 구성 카드 확인
- 운영 알림 센터에서 접수대기, 부족 재고, 진행 발주, 픽업·배송 대기 확인
- 최근 활동 로그에서 주문, 생산, 재고, 발주 변경 내역 확인
- 데모 주문이 필요하면 `open-admin-demo.cmd`로 접속 후 `데모 생성` 버튼 사용
- 시연을 다시 시작할 때는 주문관리의 `운영 데이터 초기화`로 기존 데이터를 정리

추천 멘트:
관리자 화면은 고객에게 노출하지 않고, 매장 운영자가 직접 접속해 주문 흐름을 관리하는 구조입니다. 상단의 ERP 구성 카드로 회원, 생산, 물류, 회계 기능이 어떻게 나뉘어 있는지 먼저 보여줄 수 있습니다.

### 5. ERP 4축 설명

인사/고객:
고객관리 탭에서 주문 고객 자동 집계와 고객 직접 등록, 수정, 삭제가 가능합니다.

회계:
매출관리 탭에서 상품별 매출, 원가, 이윤, 이윤율과 일자별 매출 요약을 확인할 수 있습니다. 입고완료된 발주비는 별도로 집계해 순현금흐름을 확인합니다.

물류:
픽업/배송 탭에서 매장 픽업과 배송 상담 주문을 구분하고 처리 상태를 관리합니다.

생산:
생산관리 탭에서 픽업일 기준 생산 수량과 예상 원재료를 확인하고 준비완료 상태로 일괄 처리합니다. 준비완료 처리 시 등록된 원재료 재고가 자동 차감됩니다.

### 6. 데이터 백업

- CSV 저장
- JSON 백업 저장
- JSON 백업 불러오기

추천 멘트:
아직 서버 DB 단계는 아니지만, 백업과 복원을 통해 운영 데이터 흐름을 보존할 수 있게 했습니다.

## 시연 체크리스트

- 실제 클릭 순서는 `docs/DEMO_SCENARIO.md`를 기준으로 진행합니다.
- `index.html` 메인 화면이 정상 표시되는가
- `menu.html` 메뉴 검색과 필터가 작동하는가
- 상품 상세에서 주문 문의 패널이 열리는가
- 배송 상담 선택 시 배송 주소가 필수 입력으로 바뀌는가
- 주문 문의 저장 후 `admin.html` 주문관리에서 확인되는가
- 생산관리에서 준비완료 처리가 되는가
- 재고관리에서 기본 재고 생성, 원재료 기준표 수정, 발주 후보, 생산 후 재고 차감과 사용 이력이 표시되는가
- 발주 후보를 요청으로 등록하고 발주중, 입고완료로 변경할 수 있는가
- 발주 공급처와 단가 입력 후 발주금액이 계산되는가
- 공급처를 등록·수정·삭제하고 발주 입력에서 선택할 수 있는가
- 입고완료 처리 시 현재 재고가 자동으로 증가하는가
- 매출관리에서 입고 발주비와 순현금흐름이 집계되는가
- 월별 매출, 예상이윤, 발주비, 순현금흐름 추이가 표시되는가
- 현재 기간의 월별 회계 보고서를 CSV로 저장할 수 있는가
- 픽업/배송에서 완료 처리 시 주문 상태가 픽업완료로 바뀌는가
- 매출관리에서 매출, 원가, 이윤이 계산되는가
- 고객관리에서 고객 정보가 자동 집계되는가

## 질문 대응 포인트

Q. 실제 결제도 되나요?
A. 현재 단계는 주문 문의와 관리자 ERP 흐름을 구현한 초기 모델입니다. 실제 결제는 추후 네이버 스마트스토어 또는 결제 API 연동이 필요합니다.

Q. 데이터는 어디에 저장되나요?
A. 현재는 브라우저 localStorage에 저장됩니다. 실제 운영 단계에서는 서버와 DB로 이전해야 합니다.

Q. 관리자 보안은 완성된 건가요?
A. 현재 확인 코드는 임시 접근 확인입니다. 정식 운영 전에는 서버 기반 로그인과 권한 관리가 필요합니다.

Q. ERP 4축은 어떻게 반영됐나요?
A. 관리자 첫 화면의 ERP 구성 카드와 고객관리, 생산관리, 재고관리, 픽업/배송, 매출관리 탭으로 분리해 각각의 기본 운영 기능을 구현했습니다.
