# 따뜻한 떡집 데모 시나리오

## 데모 준비

1. `open-homepage.cmd`로 메인 화면을 엽니다.
2. `open-admin-demo.cmd`로 관리자 화면을 엽니다.
3. 관리자 확인 코드 `4525`를 입력합니다.
4. 기존 데이터가 있으면 주문관리에서 `운영 데이터 초기화`를 누릅니다.
5. `데모 생성`을 눌러 샘플 주문을 추가합니다.

## 시연 흐름

### 1. 고객 첫 화면

보여줄 화면:
`index.html`

설명 포인트:
- 따뜻한 떡집 브랜드와 로고
- 메뉴와 가격 보기 버튼
- 전화 상담과 주요 메뉴 이동

짧은 멘트:
고객은 메인 화면에서 브랜드 분위기를 확인하고 메뉴 페이지로 이동할 수 있습니다.

### 2. 메뉴 확인

보여줄 화면:
`menu.html`

설명 포인트:
- BEST 상품
- 전체 메뉴 검색
- 카테고리 필터
- 메뉴 페이징

짧은 멘트:
메뉴 페이지는 메뉴판 역할과 동시에 주문 상담 요청으로 이어지는 입구입니다.

### 3. 주문 상담 요청

시연 동작:
1. BEST 상품 또는 전체 메뉴에서 상품 클릭
2. `상담 요청 남기기` 클릭
3. 수량, 희망 날짜, 시간, 연락처, 이름 입력
4. 수령 방식 선택
5. 배송 상담 선택 시 주소 입력이 필요한지 확인
6. 상담 요청 저장

짧은 멘트:
고객이 입력한 주문 상담 정보는 관리자 주문관리로 넘어가 운영 데이터가 됩니다.

### 4. 관리자 운영 흐름

보여줄 화면:
`admin.html`

설명 포인트:
- 운영 관리 대시보드
- 운영 흐름 카드
- 주문 확인 -> 생산 준비 -> 픽업/배송 -> 매출 정리

짧은 멘트:
관리자는 고객 주문을 기준으로 매장 운영 흐름을 한 화면에서 처리합니다.

### 5. 주문관리

시연 동작:
- 주문 검색
- 주문 상태 변경
- 판매가와 원가 입력
- CSV 저장 버튼 위치 확인

짧은 멘트:
주문관리에서는 접수된 주문의 기본 정보와 금액 정보를 관리합니다.

### 6. 고객관리

시연 동작:
- 자동 집계된 고객 확인
- 고객 직접 등록
- 고객 메모 입력

짧은 멘트:
고객관리는 주문 이력을 기반으로 고객별 주문 횟수와 매출을 확인할 수 있게 구성했습니다.

### 7. 생산관리

시연 동작:
- 날짜별 생산 수량 확인
- 오늘 생산 수량 확인
- 생산 항목 `준비완료` 처리

짧은 멘트:
생산관리는 실제 떡집에서 날짜별로 무엇을 얼마나 준비해야 하는지 확인하는 화면입니다.

### 8. 픽업/배송 관리

시연 동작:
- 매장 픽업과 배송 상담 주문 구분
- 물류 상태 변경
- `완료` 처리 시 주문 상태가 픽업완료로 바뀌는지 확인

짧은 멘트:
픽업과 배송 상태를 분리해 관리하면서 주문 상태와도 연결되도록 만들었습니다.

### 9. 매출관리

시연 동작:
- 총매출, 총원가, 예상이윤, 이윤율 확인
- 일자별 매출 요약 확인
- 매출 1위, 이윤 1위, 원가 미입력 건수 확인

짧은 멘트:
매출관리는 단순 주문 목록이 아니라 회계 파트처럼 이윤과 수익성을 확인할 수 있게 구성했습니다.

## 마무리 멘트

이번 프로젝트는 단순히 떡집 소개 홈페이지를 만드는 데서 끝나지 않고, 쇼핑몰 운영에 필요한 고객관리, 회계관리, 물류관리, 생산관리 흐름을 관리자 화면에 연결한 초기 ERP형 쇼핑몰입니다. 실제 운영 단계에서는 서버, 데이터베이스, 로그인 보안, 네이버 스마트스토어 연동을 추가하면 됩니다.
